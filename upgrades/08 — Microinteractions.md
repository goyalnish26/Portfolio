# 08 — Microinteractions.md

# Portfolio V3

## Microinteraction System

---

# Purpose

Microinteractions are the difference between software that feels functional and software that feels handcrafted.

Visitors rarely remember a button.

They remember how the button felt.

Every interaction should quietly communicate quality.

---

# Philosophy

Large animations create attention.

Microinteractions create trust.

The portfolio should contain hundreds of tiny interactions.

Most visitors will never consciously notice them.

That is exactly the goal.

---

# Golden Rule

Nothing should remain completely static.

Nothing should move unnecessarily.

Every interactive element should acknowledge the visitor.

Never celebrate the visitor.

---

# Response Time

Every interaction should respond immediately.

The interface should never wait.

Feedback must begin instantly.

Even if the full animation continues afterwards.

Responsiveness is perceived quality.

---

# Interaction Hierarchy

Every interaction belongs to one of four levels.

Passive

Hover

Active

Completion

Never skip levels.

---

# Hover Philosophy

Hover means

"I noticed you."

Not

"Look at me."

Hover effects should remain calm.

Tiny.

Elegant.

Predictable.

---

# Active Philosophy

Clicking confirms intention.

The interface should feel responsive.

Never explosive.

Never dramatic.

A click should feel like pressing a premium mechanical switch.

Soft resistance.

Immediate response.

Confident release.

---

# Completion Philosophy

Completion should feel satisfying.

Not celebratory.

Example

Resume download.

Instead of fireworks,

provide a subtle confirmation.

Confidence replaces excitement.

---

# Cursor Behaviour

The cursor is an invisible light source.

Not a toy.

Nearby components should politely acknowledge it.

Never chase it.

Never exaggerate movement.

---

# Magnetic Behaviour

Magnetic movement should be extremely restrained.

Maximum movement should remain minimal.

The visitor should feel attraction.

Not resistance.

If users consciously notice magnetism,

reduce it.

---

# Buttons

Buttons should communicate confidence.

Hover

Slight elevation.

Tiny lighting adjustment.

Subtle reflection.

Click

Small compression.

Natural release.

Completion

Return smoothly.

Never snap.

Never bounce.

---

# Primary Buttons

Highest emphasis.

Richer lighting.

Stronger depth.

More refined shadow.

Never larger.

Importance should come from quality,

not size.

---

# Secondary Buttons

Lower visual weight.

Calmer interaction.

Less reflection.

Smaller elevation.

---

# Ghost Buttons

Almost invisible until approached.

They quietly emerge.

Never dominate.

---

# Navigation

Navigation should never distract.

Hover

Soft highlight.

Tiny glass response.

Gentle reflection.

Section change

Smooth indicator movement.

Never jump.

Never flash.

---

# Navigation Active State

The active section should gently transition.

No hard switching.

No instant color changes.

The interface should feel aware of location.

---

# Project Cards

Projects are physical objects.

Hover

Tiny perspective.

Shadow deepens.

Reflection adjusts.

Image subtly shifts.

Technology chips acknowledge movement.

Never rotate dramatically.

Never flip.

Never spin.

---

# Project Images

Images should react independently.

Very slight parallax.

Tiny zoom.

Soft lighting.

Never become the primary animation.

Projects remain about the work.

---

# Technology Chips

Hover

Tiny lift.

Tiny brightness.

Nothing more.

Never animate all chips simultaneously.

---

# Skill Cards

Skill cards should feel intelligent.

Hover

Glass responds.

Icon slightly breathes.

Text remains stable.

Avoid floating icons.

Avoid unnecessary rotation.

---

# Timeline

Hovering a timeline entry should feel like revisiting a memory.

Background gains focus.

Typography strengthens.

Current year becomes clearer.

Previous entries remain quiet.

---

# Experience Cards

These represent professional work.

Interactions should remain restrained.

Avoid playful behaviour.

Confidence is the emotion.

---

# Contact Form

Forms deserve calmness.

Inputs should quietly respond.

Focus

Soft outline.

Tiny lighting.

Better contrast.

Errors should never shake aggressively.

Respect the user.

---

# Input Fields

Typing should feel peaceful.

Placeholder fades naturally.

Cursor remains clean.

Never animate while typing.

The user is working.

Stay out of the way.

---

# Links

Links should communicate possibility.

Underline animation should remain elegant.

Never stretch dramatically.

Never overshoot.

Subtlety builds trust.

---

# Icons

Icons should never dance.

Hover

Tiny movement.

Tiny lighting.

Tiny confidence.

Icons support interaction.

They are not decoration.

---

# Glass Panels

Hover

Reflection changes.

Depth increases.

Shadow softens.

Material acknowledges presence.

Never glow excessively.

---

# Social Icons

Visitors already know what these icons do.

Do not overdesign them.

Simple.

Elegant.

Predictable.

---

# Resume Download

This action is important.

Hover

Confidence.

Click

Acknowledgement.

Completion

Tiny confirmation.

Never intrusive.

Never modal.

---

# Theme Commands

The command itself remains unchanged.

Only the surrounding world responds.

Lighting evolves.

Glass adapts.

Particles change.

The transition should feel dimensional.

Not theatrical.

---

# Terminal

Terminal interaction is sacred.

Typing remains immediate.

Cursor remains responsive.

Commands remain untouched.

Animations must never interfere with typing speed.

Terminal performance has priority.

---

# Cards Entering View

Never fade only.

Instead

Slight upward motion.

Opacity.

Depth.

Timing.

Everything remains restrained.

---

# Statistics

Numbers should never aggressively count upward.

If animated,

they should settle naturally.

Avoid casino-style counting.

---

# Tooltips

Tooltips should appear like whispers.

Fast.

Small.

Helpful.

Never dramatic.

Never oversized.

---

# Copy Actions

Whenever content is copied,

provide quiet acknowledgement.

No popup.

No celebration.

The interface simply confirms success.

---

# Loading States

Avoid spinners whenever possible.

Use skeletons.

Progressive reveal.

Content-first loading.

Waiting should feel intentional.

---

# Empty States

Never leave empty space unexplained.

Provide calm guidance.

Never jokes.

Never unnecessary illustrations.

---

# Notifications

Notifications should almost never exist.

Prefer subtle inline confirmation.

Avoid interrupting the visitor.

---

# Cursor Leaving

When the cursor leaves an element,

the element should gracefully return home.

Never snap back.

Never instantly reset.

---

# Scroll Interaction

Hover effects should reduce during active scrolling.

Scrolling has priority.

Interactions resume naturally afterwards.

---

# Reading Awareness

While users are reading,

interactive elements become quieter.

Respect attention.

Never compete with text.

---

# Motion Budget

No interaction should exceed its emotional importance.

Small interaction.

Small motion.

Large interaction.

Measured motion.

Nothing should ever feel exaggerated.

---

# Consistency Rules

Every interaction should answer the same question.

"How would this behave if it were a physical object?"

If the answer feels artificial,

redesign it.

---

# Anti-Patterns

Never use

Aggressive scaling.

Violent shaking.

Continuous pulsing.

Rainbow gradients.

Oversized glow.

Elastic bounce.

Infinite hover loops.

Fake physics.

Long delays.

Forced attention.

The interface should feel mature.

---

# Accessibility

Every interaction must remain keyboard accessible.

Hover effects should have keyboard equivalents where appropriate.

Reduced motion should preserve interaction quality.

The portfolio should remain enjoyable without animation.

---

# Performance

Microinteractions must never block rendering.

Prefer GPU-friendly transforms.

Avoid layout shifts.

Pause expensive effects outside the viewport.

Every interaction should cost almost nothing.

---

# Definition of Success

A successful microinteraction is one the visitor never consciously notices.

They simply feel

"This interface is incredibly satisfying."

When every click,

hover,

focus,

scroll,

and interaction quietly reinforces craftsmanship,

the Microinteraction System has fulfilled its purpose.

---

# Final Principle

Visitors will forget individual interactions.

They will remember how the interface respected them.

Respect creates trust.

Trust creates memorability.

That is the purpose of every microinteraction in this portfolio.
