# Portfolio V3

## Glass, Light & Material Physics

---

# Purpose

Glass is not an effect.

Glass is a material.

It should behave exactly like a physical object.

Every glass surface should appear to have thickness.

Density.

Weight.

Temperature.

Depth.

The visitor should never consciously think

"This is glassmorphism."

Instead they should feel

"This surface actually exists."

---

# Philosophy

Glass is premium because it reacts.

Not because it is transparent.

Everything behind glass influences it.

Everything around glass influences it.

Everything above glass influences it.

Glass is constantly changing.

Even when standing still.

---

# Material Hierarchy

Every glass surface belongs to one category.

Navigation Glass

Floating Panels

Cards

Context Menus

Command Palette

Modal Windows

Quick Actions

Never invent additional glass styles.

The same material language must exist everywhere.

---

# Material Thickness

Glass should have perceived thickness.

Never appear paper thin.

Use

Edge highlights

Internal shadows

Refraction

Light absorption

Very subtle distortion

to imply thickness.

Users should almost feel the edge with their eyes.

---

# Transparency

Glass should never become invisible.

Nor should it become opaque.

Transparency should adapt to surroundings.

Busy backgrounds require stronger frosting.

Simple backgrounds require less.

Glass must always remain readable.

Readability is never negotiable.

---

# Frosting

Frost is not blur.

Frost scatters light.

Blur simply hides information.

Use frosting to create elegance.

Never increase blur simply because it looks expensive.

---

# Blur Hierarchy

Every glass object has a blur budget.

Background Glass

Very low.

Cards

Medium.

Navigation

Higher.

Dialogs

Highest.

Never exceed what is necessary.

Too much blur makes premium interfaces feel cheap.

---

# Refraction

Objects behind glass should slightly shift.

Not enough to become obvious.

Just enough for the brain to believe

there is physical material between the eye

and the object.

Refraction should remain subtle.

Anything noticeable is already too much.

---

# Internal Reflection

Glass reflects itself.

Edges should catch light.

Corners should softly glow.

The center remains calm.

Avoid identical reflections on every component.

Variation creates realism.

---

# External Reflection

The surrounding environment influences glass.

Morning

warmer reflection.

Evening

golden reflection.

Night

cool blue reflection.

Mouse movement

slight directional highlight.

Scrolling

tiny reflection drift.

Time changes

reflection temperature evolves naturally.

---

# Light Sweep

Occasionally,

light travels across glass.

Very slowly.

Very softly.

Never on every hover.

Never constantly.

Only when it feels natural.

Like sunlight moving across a desk.

---

# Light Sources

The portfolio contains invisible light sources.

Mouse.

Ambient lighting.

Time of day.

Window.

Lightning.

These influence reflections independently.

Never fake light.

Light should always originate somewhere believable.

---

# Surface Temperature

Glass should visually communicate temperature.

Morning

neutral.

Afternoon

slightly warm.

Night

cool.

Rain

colder.

Golden hour

soft amber.

Temperature changes should remain emotional,

not literal.

---

# Shadow Behaviour

Glass creates soft shadows.

Never harsh.

Shadows should become softer as glass becomes more transparent.

Floating glass should cast almost invisible shadows.

The brain notices.

The eye rarely does.

---

# Edge Lighting

Edges receive more light than the center.

Every glass object should subtly communicate this.

Avoid glowing outlines.

Instead

allow light to naturally collect along edges.

This creates premium depth.

---

# Dynamic Tint

Glass inherits tiny amounts of surrounding color.

Blue themes

slightly blue glass.

Warm themes

slightly warm glass.

Dark mode

neutral glass.

Never create colorful glass.

Tint should remain almost invisible.

---

# Mouse Influence

The cursor behaves like a moving light source.

Glass acknowledges its presence.

Tiny reflection shifts.

Tiny highlight movement.

Tiny edge brightness adjustment.

No dramatic spotlight.

No fake flashlight effect.

Subtlety wins.

---

# Scroll Influence

As scrolling continues,

glass gently changes.

Reflections lag slightly.

Highlights settle.

Tiny perspective changes occur.

The user should feel movement,

even if the glass itself barely moves.

---

# Layering

Glass exists above content,

never inside content.

Avoid stacking many glass panels.

Too many translucent layers reduce clarity.

Premium interfaces breathe.

---

# Noise

Every glass surface requires microscopic noise.

Perfect gradients feel synthetic.

Tiny imperfections create realism.

Noise should remain static.

Never animated.

Never noticeable.

---

# Water Behaviour

Rain does not simply add droplets.

Glass behaves differently.

Reflections soften.

Highlights diffuse.

Contrast lowers slightly.

Only if rain is active.

The material itself reacts.

---

# Thunder Behaviour

Lightning briefly transforms glass.

Edges brighten.

Reflections intensify.

Then everything quietly returns.

No flashing UI.

No white screen.

Only believable environmental response.

---

# Reading State

While reading,

glass becomes visually quieter.

Reflections reduce.

Movement slows.

Typography becomes the hero.

The interface politely steps aside.

---

# Focus State

Whenever an important action occurs,

glass gently supports it.

The active surface gains

slightly richer reflections,

slightly clearer contrast,

slightly stronger depth.

Never aggressive.

---

# Hover State

Hover should never activate every glass effect.

Choose

one

or

two.

Reflection.

Tiny lift.

Shadow.

Edge brightness.

Nothing more.

---

# Material Consistency

Every glass object should feel cut from the same sheet.

Not different CSS classes.

The visitor should subconsciously recognize

the same material everywhere.

---

# Performance Rules

Avoid multiple overlapping backdrop filters.

Reuse materials.

Share shaders.

Throttle updates.

Pause reflections outside viewport.

Reduce updates on battery saver.

Glass should remain beautiful,

not expensive.

---

# Mobile Adaptation

Mobile glass should simplify.

Reduce blur.

Reduce reflections.

Reduce update frequency.

Preserve identity.

Sacrifice implementation.

Never sacrifice feeling.

---

# Accessibility

Glass must never reduce readability.

Contrast always wins.

If accessibility conflicts with aesthetics,

accessibility wins.

No exceptions.

---

# Failure Behaviour

If advanced glass rendering fails,

fallback should remain elegant.

Solid translucent surfaces.

Subtle shadows.

Minimal reflections.

The experience should still feel premium.

---

# Material Integrity Test

Ask these questions.

Does this feel like one material?

Would Apple ship this?

Would Linear use this?

Would Vercel simplify this?

Would Dieter Rams remove something here?

If the answer suggests simplification,

simplify.

---

# Final Principle

The visitor should never admire the glass.

They should admire the product behind it.

Glass succeeds only when nobody notices it,

yet everything somehow feels richer because it exists.

That invisible sophistication is the definition of premium.
