## вңЁ Technical Highlights

### Glitch Transition (The Canon Break)

When switching between modes, a violent 800ms transition fires вҖ” not a fade:

```
1. RGB channel split вҖ” red/green/blue layers offset by 8вҖ“12px (200ms)
2. Pixel noise burst вҖ” CSS noise overlay
3. White flash frame вҖ” pure #FFFFFF for 80ms
4. Horizontal screen tears вҖ” 3-4 strips offset
5. New mode SNAPS in вҖ” like a different comic panel cutting in
```

### Custom Cursor

A two-layer cursor system driven entirely by `requestAnimationFrame`:

```js
// Dot: snaps to mouse instantly
cursorDot.style.left = `${e.clientX}px`

// Ring: lerps toward mouse at 12% per frame вҖ” organic feel
ringPos.x += (mousePos.x - ringPos.x) * 0.12
```

- **Dev mode** вҖ” dot `#1A1A1A`, ring `#E9C46A` at 55% opacity
- **Hacker mode** вҖ” dot `#C0C0C0`, ring `#C0C0C0` at 35% opacity
- On hover: ring scales up and fills slightly

### Architecture

| Concern | Approach |
|---|---|
| **State** | `useState` вҖ” mode, transitioning, boot index, easter egg, menu |
| **Cursor** | Pure `requestAnimationFrame` loop вҖ” zero React re-renders |
| **Scroll reveals** | `IntersectionObserver` per element |
| **Fonts** | `@import` Google Fonts inside embedded `<style>` tag |
| **CSS** | Single `CustomStyles` component injecting a `<style>` tag вҖ” no external files |
| **Build** | Vite 8 + React 19, production build in ~240ms |
